# 작문 스타일 가이드 — 템플릿

> 새 프로젝트에서 채울 템플릿. `style-guide.md`로 이름 변경 후 사용.

## 문서 구조

### 제목 계층
- **H1:** 글 제목 (1개만, 주요 키워드 포함)
- **H2:** 주요 섹션 (5-8개, 키워드 포함)
- **H3:** H2 하위 (남용 금지)
- H4 이하: 사용 안 함

### H2 개수 기준
| 글 길이 | 권장 H2 수 |
|--------|----------|
| 2,000자 이하 | 4-5개 |
| 2,000-3,000자 | 5-6개 |
| 3,000-4,000자 | 6-8개 |
| 4,000자 이상 | 7-8개 (상한) |

## 문단 구성

- 문단당 **3-5문장** (이상적 4문장)
- 문단 사이 **빈 줄 1개**
- 한 문단에 하나의 핵심 아이디어
- 도입 문단: `<도입 패턴 — 예: 공감 또는 핵심 질문>`
- 마지막 문단: `<마무리 패턴 — 예: 행동 유도 또는 요약>`

## 리스트

- 3개 이상 나열 시 사용 (2개 이하는 문장으로)
- 중첩 2단계 초과 금지
- 리스트만으로 본문 대체 금지

## 강조

- **볼드** — 핵심 키워드 첫 등장, 중요 통계
- 한 문단에 3개 초과 볼드 금지
- 이탤릭 — 외래어/전문 용어 첫 등장

## 이모지

`<이모지 정책 — 예: 사용하지 않음 / 카드 라벨에만 허용 / 자유 사용>`

## 인용 및 참고문헌

### 본문 인용 형식
- `(저자명, 연도)` 또는 `(기관명, 연도)`

### references 배열
```json
{
  "name": "기관/저자명",
  "url": "https://...",
  "type": "academic|government|industry",
  "description": "한 줄 설명"
}
```

### 인용 가능 출처 우선순위
1. `<신뢰 출처 1 — 예: 피어 리뷰 학술지>`
2. `<신뢰 출처 2 — 예: 정부 기관>`
3. `<신뢰 출처 3 — 예: 국제 전문 기관>`

## FAQ 작성

- 4-6개 (검색 의도 반영)
- 자연어 질문 형태 (`~인가요?`, `~할까요?`)
- 답변 2-4문장, 질문에 직접 답하는 문장으로 시작
- 키워드 1개 이상 자연스럽게 포함

## 메타 정보

### 메타 타이틀
- 30-60자, 키워드 앞쪽 30자 내 배치
- 패턴: `[키워드] - [부가] | <브랜드명>`

### 메타 디스크립션
- 120-155자
- 구조: 핵심 요약 + 키워드 + 행동 유도 마무리

## 숫자 표기

| 기준 | 표기 |
|-----|-----|
| 만 단위 이상 | 한글 — 1만 명, 2억 원 |
| 만 단위 미만 | 아라비아 — 500명, 3회 |
| 퍼센트 | 30%, 약 15% |

## 외래어/전문 용어

### 첫 등장 시 한글(영문) 병기
```
인지행동치료(CBT)
변증법적 행동치료(DBT)
```

### 이후 등장 — 한글만

## 도입부 (인트로)

- 길이 150-250자
- 구조:
  1. 독자 상황/감정 공감
  2. 글에서 다루는 내용 소개
  3. 읽으면 얻는 것

## 마무리 (아웃트로)

- 길이 100-200자
- 구조:
  1. 핵심 1-2문장 요약
  2. 독자 격려
  3. 행동 유도
