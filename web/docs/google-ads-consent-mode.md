# 구글 디스플레이 광고 리타게팅 & Consent Mode 정리

> 작성일: 2026-05-19 · 관련 PR: #2(Consent Mode v2) · 적용 파일: `web/app/layout.tsx`

마케팅 담당자도 이해할 수 있도록 정리한 문서입니다. 구글 디스플레이 광고의
리타게팅 동작 원리, Consent Mode 개념, 그리고 이번에 광고 효율 개선을 위해
추가한 작업을 다룹니다.

---

## 1. 구글 디스플레이 광고에서 리타게팅이 작동하는 원리

비유하면 **"가게에 들렀던 손님 손목에 도장을 찍어두고, 나중에 길에서
마주치면 다시 부르는"** 구조입니다.

1. **방문자 표시 (도장 찍기)**
   사용자가 사이트에 들어오면, 페이지에 심어둔 **구글 태그(gtag.js)** 가
   그 브라우저에 구글 광고용 식별자(쿠키)를 남깁니다.

2. **명단 만들기 (리마케팅 목록)**
   그 식별자들이 구글 광고 계정의 "이 사이트 방문자" 명단에 자동으로
   쌓입니다. (예: 가격 페이지를 봤지만 가입은 안 한 사람)

3. **다시 따라붙기 (디스플레이 노출)**
   그 사람이 나중에 다른 뉴스 사이트·블로그·YouTube를 볼 때, 구글
   디스플레이 네트워크가 명단을 알아보고 **우리 광고를 다시 노출**합니다.

4. **전환 학습 (효율의 핵심)**
   누가 광고를 다시 보고 실제로 가입/문의했는지를 태그가 구글에 알려주면,
   구글 **자동 입찰**이 "이런 사람에게 더 쓰자"를 학습해 광고비 효율이
   올라갑니다.

→ 그래서 **랜딩페이지에 태그가 박혀 있어야** 이 사이클이 돕니다.
우리 사이트는 이미 태그가 설치돼 있었고(`AW-18147654629`), 이번 작업은
그 위에서 **데이터 품질과 누락**을 개선한 것입니다.

---

## 2. Consent Mode(동의 모드)란

쿠키는 개인정보라 "동의하지 않은 사용자에게 함부로 못 쓰는" 게 원칙입니다.
선택지는 두 가지입니다.

- **태그를 아예 안 켠다** → 동의 안 한 사용자 데이터가 **0** → 구글이 그
  사람의 방문·전환을 영영 모름 → 명단·학습 데이터가 줄어 광고 효율 하락.
- **Consent Mode를 쓴다** → 동의 전에는 쿠키 없이 **익명 신호만** 보내고,
  구글이 그걸로 **전환을 통계적으로 추정(모델링)** 해 일부 복구.

즉 Consent Mode는 **데이터 손실을 줄이면서 법적 형식도 갖추는 장치**입니다.
"v2"는 여기에 `ad_user_data`(광고용 데이터 전송) /
`ad_personalization`(개인화 리마케팅) 두 항목이 추가된 최신 규격으로,
이게 없으면 구글이 리마케팅 모수를 보수적으로 깎습니다.

> **우리 정책 (A안):** 동의 배너를 운영하지 않으며, 형식상 4개 동의 항목을
> 모두 `granted`(허용)로 선언했습니다. → 법적 규격 형식은 충족하면서
> 데이터는 정상 수집. 추후 트래픽/리스크가 커지면 동의 배너 + 실제
> Consent Mode 운영(B안)으로 전환 가능.

## 3. 이번에 개선하려고 추가/수정한 작업 (PR #2)

`web/app/layout.tsx` 한 파일에 적용, `main` 머지 완료.

| 항목 | 변경 전 | 변경 후 | 효과 |
|---|---|---|---|
| **Consent Mode v2** | 없음 | `consent default` 4개 항목 `granted` 선언 스크립트 추가 (`beforeInteractive`) | 구글이 리마케팅 모수·전환 모델링을 깎지 않음 (v2 규격 충족) |
| **태그 로드 시점** | `lazyOnload` (브라우저 한가할 때, 수 초 지연 가능) | `afterInteractive` (페이지 뜬 직후) | 광고 클릭 후 빨리 이탈하는 방문자의 페이지뷰·전환 누락 방지 → 자동 입찰 학습 정확도 ↑ |
| **적용 태그 범위** | — | GA4 / Google Ads / Naver wcs / Meta Pixel 전부 | 전환·유입 데이터 정합성 통일 |
| **실행 순서** | 보장 안 됨 | 동의 기본값이 태그 설정(`config`)보다 **먼저** 실행되도록 고정 | Consent 신호 누락 방지 |

**한 줄 요약:** 리타게팅 사이클 자체는 원래 돌고 있었고, 이번엔
① 동의 규격 미비로 깎이던 데이터를 복구(Consent Mode v2)하고
② 로딩이 늦어 새던 전환·방문 데이터를 막아서(`afterInteractive`),
**구글 자동 입찰이 더 정확하게 학습 → 광고 효율 개선**되도록 한 작업입니다.
