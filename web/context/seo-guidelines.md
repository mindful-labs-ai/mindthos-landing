|---------|----------|-----|
| case-conceptualization | 2,500-4,000자 | 이론 + 사례 분석 깊이 |
| counseling-skills | 2,000-3,500자 | 실용·적용 중심 |

**기본 원칙:** 최소 1,500자, 최대 5,000자 (공백 포함, 한국어)

---

## 2. 키워드 밀도

| 키워드 유형 | 권장 밀도 | 계산 기준 |
|-----------|---------|---------|
| 주요 키워드 (Primary) | 1.0-2.0% | 전체 글자 수 대비 등장 횟수 |
| 보조 키워드 (Secondary) | 0.5-1.0% | 각 키워드별 |
| LSI / 관련 용어 | 자연스럽게 | 밀도 계산 불필요 |

**과도 삽입 금지** — 자연스러운 문맥에서 벗어난 반복은 피함. 영문 약어는 한 번 등장한 뒤에는 횟수 계산에서 빼고 자연스럽게 사용.

---

## 3. 메타 타이틀

### 형식 및 길이
- **길이:** 30-60자
- **핵심 키워드:** 앞쪽 30자 이내
- **브랜드명:** 파이프(`|`) 또는 대시(`—`) 로 구분, 선택

### 권장 패턴
```
사례개념화 5단계 — 회기 후 노트에서 가설 갱신까지
부부치료 첫 회기: 양쪽이 안전감을 느끼게 하는 구조화 스크립트
청소년상담사 자격증 로드맵 (비전공자 학점은행제 포함) | 마음토스
```

### 금지
- 키워드 나열형 ("상담 사례개념화 임상 슈퍼비전")
- 클릭베이트 ("충격적인", "당신이 모르는")
- 60자 초과

---

## 4. 메타 디스크립션

- **길이:** 120-155자
- **구조:** 핵심 요약 + 주요 키워드 + 부드러운 행동 유도
- **예시:**
  > "회기마다 사례개념화 가설을 갱신하는 5단계와 노트 템플릿을 정리했습니다. 임상 사례와 함께 바로 적용해 보세요."

### 체크리스트
- [ ] 주요 키워드 자연스럽게 포함
- [ ] 글의 핵심 가치 제안 1줄
- [ ] 120-155자
- [ ] 광고 표현 없음

---

## 5. 헤딩 구조

### H1
- 직접 사용 안 함. `posts.title` 이 자동 H1
- title 자체에 주요 키워드 포함 필수

### H2 (5-8개)
- 최소 2개의 H2 에 주요/보조 키워드 포함
- 자연스러운 문장 / 질문형 가능
- 예: `회기 안에서 사례개념화를 갱신하는 3가지 신호`

### H3
- H2 세분화 필요할 때만
- H2당 최대 3개

---

## 6. 도입부 SEO 규칙

- 주요 키워드를 **첫 100자 이내** 배치
- 검색 의도와 직결되는 핵심 문장으로 시작
- Featured Snippet 타겟 시: 정의 / 단계 / 비교를 도입부에 명시
- 글 전체 로드맵 1문장 포함 ("이 글에서는 X, Y, Z 를 다룹니다.")

---

## 7. 이미지 SEO

### alt 텍스트
- 40자 이내, 핵심 내용 설명적
- 키워드 자연 포함
- 장식 이미지 → `alt=""`

### 예시
```
alt="사례개념화 5단계 다이어그램"
alt="작업동맹 평가를 위한 SRS 척도 예시"
alt="회기 노트 자동 생성된 결과 화면 — 마음토스 축어록"
```

### 파일명
- 영문 소문자, 하이픈, 내용 설명적
- 한국어 금지
- 예: `case-conceptualization-cycle.svg`, `wai-sr-rating-scale.webp`

### Storage
- 썸네일: `blog-images/thumbnails/<slug>.webp`
- 본문 그림: `public/blog-assets/<slug>/<name>.svg|webp`

## 8. 내부 링크 전략

### 글당 내부 링크 수
- **권장:** 2-4개
- **최소:** 2개
- **최대:** 5개

### 링크 유형별 배분
| 유형 | 개수 | 비고 |
|-----|-----|-----|
| 관련 블로그 글 | 1-2개 | 같은 카테고리 또는 연관 주제 |
| 마음토스 제품/기능 | 0-1개 | InlineCTA / BottomCTA 가 주로 담당 |
| 카테고리 목록 | 0-1개 | "더 많은 ~를 보려면" |

### 앵커 텍스트
- 링크 목적지 내용을 설명적으로
- "여기 클릭", "이 글" 금지
- 키워드 자연 포함

---

## 9. 외부 링크

### 글당 1-3개
- 학술 / 정부 / 전문 기관만 인용
- `target="_blank" rel="noopener noreferrer"` — `lib/markdown/processor.ts` 가 자동 부여
- 마크다운에 그대로 `[name](url)` 작성하면 처리됨

### 허용 출처
1. 보건복지부 / 국립정신건강센터
2. 한국심리학회 / 한국상담심리학회 / 한국상담학회
3. APA, WHO, ICD, NIMH 등 국제 기관
4. 피어 리뷰 학술지 (PubMed, DBpia, KCI, ScienceDirect 등)
5. 대학·연구소 발행 자료 (서울대 / 연세대 / 카이스트 등)

## 10. URL 슬러그

### 규칙
- 영문 소문자, 하이픈 구분
- **3-5단어** 권장
- 의미 없는 단어 제거 (the, a, of 등)

### 예시
```
/blog/case-conceptualization-cycle-5-steps
/blog/working-alliance-srs-tracking
/blog/youth-counselor-credit-bank-roadmap
/blog/sat-burnout-self-care-routine
```

## 11. 구조화 데이터

### 필수 (모든 글)
1. **BlogPosting** — `lib/seo/schema.ts` 의 `generateArticleSchema()` 가 자동
2. **FAQPage** — `posts.schema_markup` 컬럼에 JSON 저장 시 자동 렌더
3. **BreadcrumbList** — `generateBreadcrumbSchema()` 자동

### 선택
- **Person** — `posts.author_id` 있으면 자동
- **VideoObject** — `posts.video_url` 있으면 `BlogPostPage` 가 자동 주입

### 질문 작성

### 답변
- 질문에 직접 답하는 문장으로 시작
- 40-300자 (스니펫 최적)

## 14. AI / GEO (Generative Engine Optimization)

상담사가 ChatGPT / Perplexity / Google AI Overviews 에서 검색할 때 마음토스 글이 인용될 수 있도록:

- 첫 문단에 **정의·답** 한 문장 포함 (passage retrieval 친화)
- 구조화된 리스트 / 표 / 단계 명시 — LLM 이 발췌하기 쉬움
- 통계 / 연구 인용은 출처와 함께 (LLM 이 신뢰도 판단)
- FAQ 가 가장 강력한 GEO 자산 — 4-5개 정확히 작성
- 마음토스 브랜드 명시는 자연스럽게 1-2회 (도배 ❌)
