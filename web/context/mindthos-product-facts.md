# 마음토스 제품 사실 자료 (Source of Truth)

마음토스 **자체 기능·보안·가격**을 다루는 글을 쓸 때, 제품 주장을 검증하는 1차 기준 문서입니다.
이 문서에 **명시된 사실만** 마음토스의 기능·정책으로 단정할 수 있습니다. 여기 없는 기능을
"마음토스가 제공한다"고 쓰지 마세요. 추측·일반론을 자사 기능으로 둔갑시키면 자사 서비스를
잘못 소개하는 치명적 오류가 됩니다.

> 근거: `mindthos-web` 소스 코드(`/Users/sg-mini/Documents/GitHub/mindthos-web/src`),
> 랜딩 가격 섹션(`web/components/home/sections/PricingSection.tsx`), 공개 `/security` 정책.
> 사실이 바뀌면 이 문서를 먼저 갱신합니다(코드/페이지가 단일 진실 원본).

## 1. 검증된 기능 (소스에 실재 — 글에 사용 가능)

- **축어록 자동화** — 회기 녹음을 화자 분리된 발화 단위 텍스트로 변환. 일반/고급 축어록 선택.
- **화자 분리(speaker diarization)** — 상담사·내담자 등 화자별 라벨링. 비언어 표현(침묵·한숨 등) 마커.
- **진행기록(상담노트) 작성 보조** — 회기 기반 노트 초안 생성.
- **사례개념화 보조** — CBT 등 이론별 사례개념화 항목 구성.
- **제노그램 생성** — 가족체계 도식 생성.
- **AI 슈퍼비전 보조** — 회기 자료 기반 자기 슈퍼비전 지원.
- **비식별화** — 식별정보 비식별 처리 기능.
- **기관 양식 맞춤 템플릿** — 가족센터·EAP 보고서 등 기관 서식 템플릿(검증된 범위 내).
- 모든 자동 생성물은 **임상가 최종 확인을 전제로 한 초안**입니다(단정 금지).

## 2. 검증된 가격 (PricingSection.tsx — 갱신 가능, 단정 시 "변동 가능" 명시)

**크레딧 기반 월 구독**. 작업 종류·기록 길이에 따라 크레딧 소모.

| 플랜 | 월 요금 | 월 크레딧 | 대상 |
| --- | --- | --- | --- |
| 스타터 | 8,900원 | 500 | 처음 써보는 개인 상담사 |
| 플러스 | 29,900원 | 2,500 | 꾸준히 운영하는 개인 상담사 |
| 프로 | 49,900원 | 5,000 | 풀타임 상담사·소규모 팀 |
| 기관 | 문의(맞춤 크레딧) | 협의 | 센터·병원·공공기관·대학 |

- 정확한 금액·크레딧은 변동될 수 있으므로 "공식 가격 안내 확인" 문구를 덧붙입니다.
- **시트(seat) 단위 과금 아님.** "상담사 수 × 단가"로 단정하지 마세요. 기관은 맞춤 크레딧·협의.

## 3. 검증된 보안 정책 (공개 `/security`)

- 내담자 데이터 **모델 학습 미사용**(외부 모델 제공자에도 학습 미사용 적용).
- **저장 시 암호화·전송 구간 암호화**.
- **접근 권한 분리·비식별화**.
- 본인 삭제·삭제 요청 처리.

"정책 확정 후 공개" / "도입 시 별도 안내" 항목(구체 알고리즘·키 관리·로테이션·보관 기간·인증
취득)은 **확정 사실로 단정하지 말고** 그 표현 그대로 기재합니다.

## 4. ❌ 존재하지 않는 기능 (deny-list — 자사 기능으로 절대 쓰지 말 것)

소스·공개 정책에 **없습니다**. 이 항목을 "마음토스가 지원/제공/구성한다"고 쓰면 사실 오류입니다.

- **SSO / SAML / OIDC / OAuth 통합 인증 / 싱글 사인온**
- **SLA 보장 / 가동률(99.9% 등) 약정**
- **EMR / 전자의무기록 연동**
- **RBAC / 역할 기반 접근제어 / 권한 매트릭스 / 세분화된 관리자 권한 체계**
- **감사 로그(audit log) / 접근 로그 변조 방지 기능**
- **시트(seat) 단위 과금 / 좌석 라이선스**
- **무료 체험(free trial) 플랜**
- **데이터 자동 파기 옵션 / 저장 전 암호화 옵션 / 다단계 보관기간 선택**
- **바우처 전용 서식 템플릿** (기관 양식 템플릿은 가족센터·EAP 등 검증 범위만)
- **표준 DPA / SLA 문서 제공** (위탁·보안 협의는 "도입 상담 시 협의"로만 표현)

## 5. 글 작성 규칙

1. 위 deny-list 주제가 **글의 핵심 전제**이면(예: "마음토스 SSO 연동 가이드"), 그 기능이
   마음토스에 있다고 전제하는 글을 쓰지 마세요. 일반 개념 설명은 가능하되 **"마음토스가
   제공한다"는 서술을 넣지 말고**, 검증된 인접 기능(기관 플랜 계정 관리 등 §1)으로
   안전하게 전환하거나, 자사 기능 단정 없이 "기관 검토 항목" 관점으로만 다룹니다.
2. 기관 플랜은 "여러 상담사 계정·권한 관리, 기관 양식 맞춤 템플릿, 관리자 도구/팀 관리,
   도입 지원·보안 협의"까지만 단정합니다(이 표현 범위 밖은 "협의").
3. 보안·키 관리·인증·보관 기간은 §3 표현을 그대로 사용, "도입 시 별도 안내" 프레이밍 유지.
4. 가격은 §2 표를 기준으로 쓰되 "변동 가능, 공식 안내 확인" 문구 필수.
