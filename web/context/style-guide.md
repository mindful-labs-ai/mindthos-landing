# 마음토스 블로그 스타일 가이드

## 1. 문서 구조

### 제목 계층
- **H1:** 본문에 직접 쓰지 않음. `page.tsx` 가 `posts.title` 을 H1 으로 렌더
- **H2:** 주요 섹션 구분 (5-8개 권장, 주요 키워드 자연스럽게 포함)
- **H3:** H2 하위 세부 항목 (필요 시에만)
- H4 이하: 사용하지 않음

### H2 개수 기준
| 글 길이 | 권장 H2 수 |
|--------|----------|
| 1,800자 이하 | 4-5개 |
| 2,000-3,000자 | 5-6개 |
| 3,000-4,000자 | 6-8개 |
| 4,000자 이상 | 7-8개 (상한) |

---

## 2. 문단 구성

- 문단당 **3-5문장** (이상적 4문장)
- 문단 사이 빈 줄 1개로 구분
- 한 문단에 하나의 핵심 아이디어
- 도입 문단: 임상 현장의 구체적 장면 / 통계 / 핵심 질문으로 시작
- 마지막 문단: 행동 유도(부드럽게) 또는 핵심 요약

---

## 3. 리스트 사용 규칙

### 불릿 (`-`)
- 3개 이상 나열, 순서 무관
- 각 항목 1-2줄
- 리스트 앞에 안내 문장 필요

### 번호 리스트 (`1.`)
- 절차 / 단계 / 순서가 중요한 항목
- 사례 분석, 회기 진행 순서 등

### 금지
- 2개 이하 리스트 사용 (문장으로 풀어 쓸 것)
- 중첩 2단계 초과
- 리스트만으로 본문 대체

---

## 4. 강조

### 볼드 `**텍스트**`
- 핵심 키워드 첫 등장
- 통계·수치·정의 강조
- 한 문단 최대 2개

### 이탤릭
- 외래어 / 영문 용어 첫 등장 (`*motivational interviewing*`)
- 인용문 표기

### 금지
- 문단 전체 볼드
- 장식 목적 볼드

---

## 5. 이모지

**사용하지 않음.** 본문·제목·FAQ·메타 어디에도 이모지 ❌. 마음토스 디자인 시스템 규칙(`web/CLAUDE.md`)을 따름.

## 6. 인용 및 참고문헌

### 본문 인용 형식
- 학술/이론 인용: `(저자, 연도)` — `(Beck, 1979)`, `(WHO, 2023)`
- 직접 인용: 큰따옴표 + 출처 — "치료 동맹은 변화의 가장 강력한 단일 예측변수다"(Norcross & Lambert, 2018)

### references 배열 구성
```json
{
  "references": [
    {
      "name": "기관/저자명",
      "url": "https://...",
      "type": "academic | government | industry",
      "description": "(선택) 짧은 설명"
    }
  ]
}
```

### 출처 우선순위
1. APA, WHO, ICD 등 국제 임상 기관
2. 보건복지부 / 국립정신건강센터 / 한국보건사회연구원
3. 한국심리학회 / 한국상담심리학회 / 한국상담학회
4. 피어 리뷰 학술지 (PubMed, DBpia, KCI)
5. 마음토스 자체 기술 / 보안 문서 (tech-blog 한정)

## 7. FAQ 작성 기준

### 구성
- **개수:** 3-5개 (4-5 권장)
- **형식:** 상담사가 실제로 물어볼 만한 자연어 질문

### 질문 패턴
- "~인가요? / ~할 수 있나요?"
- "~할 때는 어떻게 해야 하나요?"
- "~와 ~의 차이는 무엇인가요?"
- "마음토스에서 ~를 어떻게 다루나요?" (제품 관련 글에서만)

### 답변 기준
- 100-250자, 2-4문장
- 질문에 직접 답하는 문장으로 시작
- 본문에 근거가 있는 내용만 (새 사실 추가 금지)
- 답변 끝에 광고 / CTA 금지

## 8. 메타 정보

### 메타 타이틀
- **길이:** 30-60자 (한국어)
- **형식:** 핵심 키워드 앞쪽, 보조 문구 뒤
- **예시 패턴:**
  - `사례개념화 5단계: 임상에서 가설 갱신하는 법`
  - `상담사 EAP 도입 가이드 — 마음토스`
  - `청소년상담사 자격증 로드맵 (비전공자 포함)`

### 메타 디스크립션
- **길이:** 120-155자
- **포함:** 주요 키워드 + 글의 가치 제안 + 행동 유도
- **예시:**
  > "사례개념화 가설을 회기마다 갱신하는 실무 5단계를 정리했습니다. 현장 사례와 함께 적용 방법을 확인해 보세요."

## 9. 숫자 표기

| 기준 | 표기 |
|-----|-----|
| 만 단위 이상 | 한글 — 1만 명, 3억 원 |

## 10. 임상·학술 용어

### 첫 등장 시 한글 + 영문 / 약어 병기
```
인지행동치료(CBT)
변증법적 행동치료(DBT)
수용전념치료(ACT)
동기강화상담(motivational interviewing, MI)
외상 후 스트레스 장애(PTSD)
```

### 이후 등장
- 한글 또는 약어 (CBT, DBT) 선택
- 한 글 안에서 일관되게 사용

### 한국어 표준
- Working alliance → 작업동맹 (치료동맹 병용 가능)
- Case conceptualization → 사례개념화
- Supervision → 슈퍼비전 (수퍼비전 X)
- Burnout → 소진 (번아웃 병용 가능)
- Rapport → 라포 (라포어 X)
- Countertransference → 역전이

## 11. 도입부 (인트로)

- **길이:** 150-250자
- **구조:**
  1. 임상 현장의 구체적 장면 / 통계 / 독자의 고민
  2. 이 글에서 다룰 내용 미리보기
  3. 읽으면 얻는 것 (체크리스트, 스크립트, 결정 트리 등)
- 주요 키워드를 첫 100자에 자연스럽게 포함
- 질문형 시작 허용 ("회기 직후 노트를 다시 쓰는 데 1시간씩 걸리신 적 있으신가요?")

---

## 12. 마무리 (아웃트로)

- **길이:** 100-200자
- **구조:**
  1. 핵심 1-2문장 요약
  2. 동료 상담사를 향한 짧은 격려 / 공감
  3. 행동 유도는 부드럽게 — CTA 컴포넌트가 본격적 전환을 담당

좋은 예: "기록 시간을 줄인 만큼, 자기 슈퍼비전과 자기돌봄에 더 깊이 들어갈 수 있기를 바랍니다."
